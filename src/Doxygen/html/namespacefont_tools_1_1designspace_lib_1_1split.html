Detailed Description

Allows building all the variable fonts of a DesignSpace version 5 by
splitting the document into interpolable sub-space, then into each VF.

366 # What rules to keep:
367 # - Keep the rule if any conditionset is relevant.
368 # - A conditionset is relevant if all conditions are relevant or it is empty.
369 # - A condition is relevant if
370 # - axis is point (C-AP),
371 # - and point in condition's range (C-AP-in)
372 # (in this case remove the condition because it's always true)
373 # - else (C-AP-out) whole conditionset can be discarded (condition false
374 # => conditionset false)
375 # - axis is range (C-AR),
376 # - (C-AR-all) and axis range fully contained in condition range: we can
377 # scrap the condition because it's always true
378 # - (C-AR-inter) and intersection(axis range, condition range) not empty:
379 # keep the condition with the smaller range (= intersection)
380 # - (C-AR-none) else, whole conditionset can be discarded
381 newRules: List[RuleDescriptor] = []
382 for rule in rules:
383 newRule: RuleDescriptor = RuleDescriptor(
384 name=rule.name, conditionSets=[], subs=rule.subs
385 )
386 for conditionset in rule.conditionSets:
387 cs = _conditionSetFrom(conditionset)
388 newConditionset: List[Dict[str, Any]] = []
389 discardConditionset = False
390 for selectionName, selectionValue in designRegion.items():
391 # TODO: Ensure that all(key in conditionset for key in region.keys())?
392 if selectionName not in cs:
393 # raise Exception("Selection has different axes than the rules")
394 continue
395 if isinstance(selectionValue, (float, int)): # is point
396 # Case C-AP-in
397 if selectionValue in cs[selectionName]:
398 pass # always matches, conditionset can stay empty for this one.
399 # Case C-AP-out
400 else:
401 discardConditionset = True
402 else: # is range
403 # Case C-AR-all
404 if selectionValue in cs[selectionName]:
405 pass # always matches, conditionset can stay empty for this one.
406 else:
407 intersection = cs[selectionName].intersection(selectionValue)
408 # Case C-AR-inter
409 if intersection is not None:
410 newConditionset.append(
411 {
412 "name": selectionName,
413 "minimum": intersection.minimum,
414 "maximum": intersection.maximum,
415 }
416 )
417 # Case C-AR-none
418 else:
419 discardConditionset = True
420 if not discardConditionset:
421 newRule.conditionSets.append(newConditionset)
422 if newRule.conditionSets:
423 newRules.append(newRule)
424

Split the given DS5 into several interpolable sub-designspaces.
There are as many interpolable sub-spaces as there are combinations of
discrete axis values.

E.g. with axes:
    - italic (discrete) Upright or Italic
    - style (discrete) Sans or Serif
    - weight (continuous) 100 to 900

There are 4 sub-spaces in which the Weight axis should interpolate:
(Upright, Sans), (Upright, Serif), (Italic, Sans) and (Italic, Serif).

The sub-designspaces still include the full axis definitions and STAT data,
but the rules, sources, variable fonts, instances are trimmed down to only
keep what falls within the interpolable sub-space.

Args:
  - ``makeNames``: Whether to compute the instance family and style
    names using the STAT data.
  - ``expandLocations``: Whether to turn all locations into "full"
    locations, including implicit default axis values where missing.
  - ``makeInstanceFilename``: Callable to synthesize an instance filename
    when makeNames=True, for instances that don't specify an instance name
    in the designspace. This part of the name generation can be overridden
    because it's not specified by the STAT table.

.. versionadded:: 5.0
