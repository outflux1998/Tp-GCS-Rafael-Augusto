151 # The buffer we got is byte oriented, and a character may span in more than one
152 # buffers. In case the last one or two byte in last buffer is not
153 # complete, we record how many byte needed to complete that character
154 # and skip these bytes here. We can choose to record those bytes as
155 # well and analyse the character once it is complete, but since a
156 # character will not make much difference, by simply skipping
