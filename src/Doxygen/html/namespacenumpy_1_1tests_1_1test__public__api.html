115# Historically NumPy has not used leading underscores for private submodules
116# much. This has resulted in lots of things that look like public modules
117# (i.e. things that can be imported as `import numpy.somesubmodule.somefile`),
118# but were never intended to be public. The PUBLIC_MODULES list contains
119# modules that are either public because they were meant to be, or because they
120# contain public functions/objects that aren't present in any other namespace
121# for whatever reason and therefore should be treated as public.
122#
123# The PRIVATE_BUT_PRESENT_MODULES list contains modules that look public (lack
124# of underscores) but should not be used. For many of those modules the
125# current status is fine. For others it may make sense to work on making them
126# private, to clean up our public API and avoid confusion.
