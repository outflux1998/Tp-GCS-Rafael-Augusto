Detailed Description

Python wrapper for LDA using `MALLET <http://mallet.cs.umass.edu/>`_.

Communication between MALLET and Python takes place by passing around data files on disk
and calling Java with subprocess.call().

Warnings
--------
This is **only** python wrapper for `MALLET LDA <http://mallet.cs.umass.edu/>`_,
you need to install original implementation first and pass the path to binary to ``mallet_path``.

484 """Get document topic vectors from MALLET's "doc-topics" format, as sparse gensim vectors.
485
486 Parameters
487 ----------
488 fname : str
489 Path to input file with document topics.
490 eps : float, optional
491 Threshold for probabilities.
492 renorm : bool, optional
493 If True - explicitly re-normalize distribution.
494
495 Raises
496 ------
497 RuntimeError
498 If any line in invalid format.
499
500 Yields
501 ------
502 list of (int, float)
503 LDA vectors for document.
504
505 """
506 mallet_version = self.get_version(self.mallet_path)
507 with utils.open(fname, 'rb') as fin:
508 for lineno, line in enumerate(fin):
509 if lineno == 0 and line.startswith(b"#doc "):
510 continue # skip the header line if it exists
511
512 parts = line.split()[2:] # skip "doc" and "source" columns
513
514 # the MALLET doctopic format changed in 2.0.8 to exclude the id,
515 # this handles the file differently dependent on the pattern
516 if len(parts) == 2 * self.num_topics:
517 doc = [
518 (int(id_), float(weight)) for id_, weight in zip(*[iter(parts)] * 2)
519 if abs(float(weight)) > eps
520 ]
521 elif len(parts) == self.num_topics and mallet_version != '2.0.7':
522 doc = [(id_, float(weight)) for id_, weight in enumerate(parts) if abs(float(weight)) > eps]
523 else:
524 if mallet_version == "2.0.7":
525 """
526
527 1 1 0 1.0780612802674239 30.005575655428533364 2 0.005575655428533364
528 2 2 0 0.9184413079632608 40.009062076892971008 3 0.009062076892971008
529 In the above example there is a mix of the above if and elif statement.
530 There are neither `2*num_topics` nor `num_topics` elements.
531 It has 2 formats 40.009062076892971008 and 0 1.0780612802674239
532 which cannot be handled by above if elif.
533 Also, there are some topics are missing(meaning that the topic is not there)
534 which is another reason why the above if elif fails even when the `mallet`
535 produces the right results
536
537 """
538 count = 0
539 doc = []
540 if len(parts) > 0:
541 while count < len(parts):
542 """
543 if section is to deal with formats of type 2 0.034
544 so if count reaches index of 2 and since int(2) == float(2) so if block is executed
545 now there is one extra element afer 2, so count + 1 access should not give an error
546
547 else section handles formats of type 20.034
548 now count is there on index of 20.034 since float(20.034) != int(20.034) so else block
549 is executed
550
551 """
552 if float(parts[count]) == int(parts[count]):
553 if float(parts[count + 1]) > eps:
554 doc.append((int(parts[count]), float(parts[count + 1])))
555 count += 2
556 else:
557 if float(parts[count]) - int(parts[count]) > eps:
558 doc.append((int(parts[count]) % 10, float(parts[count]) - int(parts[count])))
559 count += 1
560 else:
561 raise RuntimeError("invalid doc topics format at line %i in %s" % (lineno + 1, fname))
562
563 if renorm:
564 # explicitly normalize weights to sum up to 1.0, just to be sure...
565 total_weight = float(sum(weight for _, weight in doc))
566 if total_weight:
