33# Historically SciPy has not used leading underscores for private submodules
34# much. This has resulted in lots of things that look like public modules
35# (i.e. things that can be imported as `import scipy.somesubmodule.somefile`),
36# but were never intended to be public. The PUBLIC_MODULES list contains
