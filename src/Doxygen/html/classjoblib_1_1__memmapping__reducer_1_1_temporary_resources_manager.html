Detailed Description

Stateful object able to manage temporary folder and pickles

It exposes:
- a per-context folder name resolving API that memmap-based reducers will
  rely on to know where to pickle the temporary memmaps
- a temporary file/folder management API that internally uses the
  resource_tracker.

589 # to call terminate explicitly: note we do not pass any reference to
590 # ensure that this callback won't prevent garbage collection of
591 # parallel instance and related file handler resources such as POSIX
592 # semaphores and pipes

598 # None just before exiting when accessing the delete_folder
599 # function from the closure namespace. So instead we reimport
600 # the delete_folder function explicitly.

604 # easy vendoring.
